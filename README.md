# 一会儿吃什么？🍽️

帮助快速决定今天吃什么的离线 PWA 应用。基于用户偏好、历史记录、场景模式进行智能推荐。

## 功能

| 功能 | 说明 |
|------|------|
| **智能推荐** | 基于场景 + 口味偏好 + 时段 + 历史记录的多维度评分算法 |
| **场景模式** | 随便吃/想吃爽/健康/减肥/省钱/很累/聚餐 7 种场景 |
| **分类筛选** | 点选分类限定推荐范围，支持多选 |
| **餐品管理** | 新增 / 编辑 / 删除 / 批量标签 / JSON 导入导出 |
| **历史记录** | 按日期分组展示，自动降低近期餐品权重 |
| **提示语系统** | 温柔 / 搞笑 / 毒舌 三种风格 |
| **深色模式** | 跟随系统或手动切换 |
| **离线可用** | 所有数据存储在浏览器 IndexedDB |

## 快速开始

直接打开 [eat.html](./eat.html) 即可使用，无需服务器。

### iPhone 添加主屏幕

1. Safari 打开应用
2. 点分享按钮 → **添加到主屏幕**
3. 之后像原生 App 一样离线使用

## 数据

- 内置 **120 种**常见餐品
- 覆盖 7 个分类：中餐 / 火锅 / 快餐 / 面食 / 日料 / 韩餐 / 简餐
- 22 个口味标签

## 技术栈

- 纯 HTML + CSS + JavaScript
- IndexedDB 本地存储
- Service Worker 离线缓存
- 单文件架构（全部内联，零依赖）
- iOS 风格 UI

## 项目结构

```
/food
├── eat.html          # 单文件应用（推荐使用）
├── index.html        # 多文件版
├── manifest.json     # PWA 配置
├── service-worker.js # 离线缓存
├── data/             # JSON 数据源
├── js/               # JS 逻辑
└── css/              # 样式
```

## 推荐算法

评分 = 基础权重 + 场景匹配 + 口味匹配 + 时段匹配 + 价格适配 + 近期惩罚 + 随机扰动

- **近期惩罚**：3 天内吃过的餐品降低分数
- **随机扰动**：防止推荐结果过于固定
